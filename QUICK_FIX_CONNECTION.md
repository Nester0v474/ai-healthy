# Быстрое решение проблемы подключения

## Проблема:
Ошибка: `failed to connect to /10.0.2.2 (port 3000)`

Это означает, что вы используете **реальное устройство**, но в коде указан адрес для **эмулятора**.

## Решение:

### Шаг 1: Узнайте IP адрес вашего компьютера

**Windows:**
1. Откройте командную строку (Win + R → `cmd`)
2. Введите: `ipconfig`
3. Найдите "IPv4 Address" в разделе вашего Wi-Fi адаптера
4. Это будет что-то вроде `192.168.0.XXX` или `192.168.1.XXX`

**Mac/Linux:**
```bash
ifconfig | grep inet
```

### Шаг 2: Обновите BACKEND_URL

Откройте файл `app/build.gradle.kts` и найдите строку:
```kotlin
buildConfigField("String", "BACKEND_URL", "\"http://192.168.0.100:3000/\"")
```

Замените `192.168.0.100` на ваш реальный IP адрес, например:
```kotlin
buildConfigField("String", "BACKEND_URL", "\"http://192.168.0.105:3000/\"")
```

### Шаг 3: Убедитесь, что сервер запущен

1. Откройте терминал в папке `backend`
2. Запустите сервер:
   ```bash
   npm start
   ```
3. Вы должны увидеть:
   ```
   Server is running on port 3000
   Server listening on: http://0.0.0.0:3000
   ```

### Шаг 4: Проверьте подключение

1. Убедитесь, что телефон и компьютер в **одной Wi-Fi сети**
2. В браузере телефона откройте: `http://YOUR_IP:3000/health`
3. Должен вернуться JSON: `{"status":"ok","timestamp":"..."}`

### Шаг 5: Пересоберите проект

1. Build → Rebuild Project
2. Запустите приложение заново

## Важно:

- ✅ Телефон и компьютер должны быть в **одной Wi-Fi сети**
- ✅ Сервер должен быть **запущен** перед запуском приложения
- ✅ Файрвол может блокировать подключения - проверьте настройки
- ✅ Используйте правильный IP адрес (не `10.0.2.2` для реального устройства)

## Альтернативное решение (для тестирования):

Если не хотите настраивать локальную сеть, можете использовать ngrok:

1. Установите ngrok: https://ngrok.com/
2. Запустите: `ngrok http 3000`
3. Используйте полученный URL (например, `https://abc123.ngrok.io`) в `BACKEND_URL`
